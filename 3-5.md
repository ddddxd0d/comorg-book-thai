# 3.5 การเชื่อมต่อแบบจุดต่อจุด (Point-to-Point Interconnect)

----

สถาปัตยกรรมบัสแบบใช้ร่วมกัน (Shared bus architecture) เคยเป็นวิธีการมาตรฐานในการเชื่อมต่อระหว่างตัวประมวลผลและส่วนประกอบอื่นๆ (หน่วยความจำ, I/O และอื่นๆ) มานานหลายทศวรรษ แต่ระบบในปัจจุบันหันมาพึ่งพาการเชื่อมต่อแบบจุดต่อจุดมากขึ้นแทนการใช้บัสร่วมกัน

เหตุผลสำคัญที่ผลักดันให้เกิดการเปลี่ยนแปลงจากบัสไปสู่การเชื่อมต่อแบบจุดต่อจุด คือข้อจำกัดทางไฟฟ้าที่พบเมื่อต้องเพิ่มความถี่ของบัสแบบขนานที่กว้างขึ้น (wide synchronous buses) เมื่ออัตราการส่งข้อมูลสูงขึ้นเรื่อยๆ การประสานเวลา (synchronization) และการทำหน้าที่ตัดสินสิทธิ์การใช้บัส (arbitration) ให้ทันท่วงทีจึงทำได้ยากขึ้นอย่างยิ่ง นอกจากนี้ ด้วยการมาถึงของชิปหลายคอร์ (multicore chips) ที่มีตัวประมวลผลหลายตัวและหน่วยความจำจำนวนมากรวมอยู่ในชิปเดียว พบว่าการใช้บัสร่วมกันแบบเดิมยิ่งเพิ่มความยากลำบากในการเพิ่มอัตราข้อมูลของบัสและลดความหน่วง (latency) เพื่อให้ก้าวทันความเร็วของตัวประมวลผล เมื่อเทียบกับบัสแบบใช้ร่วมกันแล้ว การเชื่อมต่อแบบจุดต่อจุดมีความหน่วงต่ำกว่า มีอัตราข้อมูลสูงกว่า และมีความสามารถในการขยายระบบ (scalability) ที่ดีกว่า

ในหัวข้อนี้ เราจะพิจารณาตัวอย่างที่สำคัญและเป็นตัวแทนของแนวทางการเชื่อมต่อแบบจุดต่อจุด นั่นคือ QuickPath Interconnect (QPI) ของ Intel ซึ่งเปิดตัวในปี 2008
## คุณลักษณะสำคัญของ QPI และแผนผังการเชื่อมต่อแบบจุดต่อจุดอื่นๆ
* การเชื่อมต่อตรงแบบหลายช่องทาง (Multiple direct connections): ส่วนประกอบต่างๆ ภายในระบบมีการเชื่อมต่อตรงแบบคู่ขนานไปยังส่วนประกอบอื่นๆ ซึ่งช่วยกำจัดความจำเป็นในการตัดสินสิทธิ์ (arbitration) ที่พบในระบบส่งข้อมูลแบบใช้ตัวกลางร่วมกัน
* สถาปัตยกรรมโปรโตคอลแบบเลเยอร์ (Layered protocol architecture): เช่นเดียวกับที่พบในสภาพแวดล้อมเครือข่าย (เช่น เครือข่ายข้อมูลฐาน TCP/IP) การเชื่อมต่อระดับตัวประมวลผลเหล่านี้ใช้สถาปัตยกรรมโปรโตคอลแบบเลเยอร์ แทนที่จะใช้เพียงสัญญาณควบคุมง่ายๆ แบบที่พบในระบบบัส
* การถ่ายโอนข้อมูลแบบแพ็กเก็ต (Packetized data transfer): ข้อมูลจะไม่ถูกส่งเป็นกระแสบิตดิบ (raw bit stream) แต่จะถูกส่งเป็นลำดับของแพ็กเก็ต ซึ่งแต่ละแพ็กเก็ตจะประกอบด้วยส่วนหัวควบคุม (control headers) และรหัสควบคุมข้อผิดพลาด (error control codes)

ในระบบสมัยใหม่ การเชื่อมต่อจาก IOH (I/O Hub) ไปยังตัวควบคุมอุปกรณ์ I/O จะใช้เทคโนโลยีการเชื่อมต่อที่เรียกว่า PCI Express (PCIe) ซึ่งจะกล่าวถึงในภายหลังของบทนี้ โดย IOH จะทำหน้าที่แปลโปรโตคอลและรูปแบบข้อมูลระหว่าง QPI และ PCIe นอกจากนี้ คอร์ของตัวประมวลผลยังเชื่อมต่อกับโมดูลหน่วยความจำหลัก (ซึ่งมักใช้เทคโนโลยี DRAM) โดยใช้ บัสหน่วยความจำเฉพาะ (dedicated memory bus)

สถาปัตยกรรมโปรโตคอล 4 เลเยอร์ของ QP
* เลเยอร์กายภาพ (Physical Layer): ประกอบด้วยสายสัญญาณจริงที่ใช้ส่งสัญญาณ รวมถึงวงจรและตรรกะที่รองรับคุณสมบัติเสริมในการส่งและรับสัญญาณ 0 และ 1 หน่วยของการถ่ายโอนข้อมูลในเลเยอร์นี้คือ 20 บิต เรียกว่า Phit (Physical Unit)
* เลเยอร์การเชื่อมต่อ (Link Layer): รับผิดชอบในการส่งข้อมูลอย่างเชื่อถือได้และการควบคุมการไหลของข้อมูล (flow control) หน่วยของการถ่ายโอนข้อมูลในเลเยอร์นี้คือ 80 บิต เรียกว่า Flit (Flow Control Unit)
* เลเยอร์การหาเส้นทาง (Routing Layer): กำหนดกรอบการทำงานสำหรับการส่งแพ็กเก็ตข้อมูลผ่านโครงสร้างการเชื่อมต่อ (fabric)
* เลเยอร์โปรโตคอล (Protocol Layer): เป็นชุดกฎเกณฑ์ระดับสูงสำหรับการแลกเปลี่ยน แพ็กเก็ต (Packets) ข้อมูลระหว่างอุปกรณ์ โดยหนึ่งแพ็กเก็ตจะประกอบด้วยจำนวน Flit ที่เป็นตัวเลขเต็มหน่วย
