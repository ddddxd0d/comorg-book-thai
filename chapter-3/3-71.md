# 3.7 คำศัพท์สำคัญ, คำถามทบทวนเนื้อหา และแบบฝึกหัด

## คำถามทบทวนเนื้อหา (Review Questions)

### 3.1 หมวดหมู่คำสั่งคอมพิวเตอร์
- Data Processing: การประมวลผลข้อมูล เช่น บวก ลบ AND OR
- Data Storage: การเก็บและโหลดข้อมูล
- Data Movement: การเคลื่อนย้ายข้อมูล (I/O)
- Control: การควบคุมลำดับการทำงาน เช่น jump, branch

### 3.2 สถานะของการทำงานคำสั่ง
- Fetch: ดึงคำสั่งจากหน่วยความจำ
- Decode: แปลคำสั่ง
- Execute: ดำเนินการ
- Store (Write-back): เก็บผลลัพธ์

### 3.3 การจัดการ Multiple Interrupts
- Disable Interrupts: ปิด interrupt อื่นชั่วคราว
- Priority Interrupts: จัดลำดับความสำคัญ

### 3.4 รูปแบบการถ่ายโอนข้อมูลของบัส
- Memory → CPU
- CPU → Memory
- I/O → CPU
- CPU → I/O
- I/O ↔ Memory (DMA)

### 3.5 QPI Protocol Layers
- Physical Layer
- Link Layer
- Routing Layer
- Protocol Layer

### 3.6 PCIe Protocol Layers
- Physical Layer
- Data Link Layer
- Transaction Layer

## โจทย์ปัญหา (Problems)

### 3.1 การทำงานโปรแกรม

Step 1: Load AC จาก device 5 → AC = 3  
Step 2: Add memory[940] (2) → AC = 5  
Step 3: Store AC ไป device 6  

ผลลัพธ์: AC = 5

---

### 3.2 การใช้ MAR และ MBR
- MAR: เก็บ address
- MBR: เก็บข้อมูล

ขั้นตอน:
1. MAR ← address
2. MBR ← memory
3. IR ← MBR
4. Decode
5. Execute
6. Store result

---

### 3.3 ไมโครโพรเซสเซอร์ 32-bit

ก. Memory สูงสุด  
Address = 24 bit → 2^24 = 16 MB  

ข. ผลต่อความเร็ว  
1. 32-bit address / 16-bit data → ช้าลง  
2. 16-bit address / 16-bit data → ช้ากว่า  

ค. ขนาด register  
PC = 24 bit  
IR = 32 bit  

---

### 3.4 ระบบ 16-bit

ก. 2^16 = 64K words  
ข. 64 KB (ถ้า memory 8-bit)  
ค. ใช้ Isolated I/O  
ง.  
- 8-bit port = 256 ports  
- 16-bit port = 128 ports  

---

### 3.5 Data Transfer Rate

Clock = 8 MHz  
Bus cycle = 4 clocks  
Data = 2 bytes  

Rate = 4 MB/s  

เพิ่ม bus หรือ clock → ได้ผลเท่ากัน (2 เท่า)

---

### 3.6 I/O กับ Teletype

Input:
- keyboard → INPR
- FGI = 1 → CPU อ่าน

Output:
- CPU → OUTR
- FGO = 0 → ส่งออก

ใช้ IEN:
- ทำ interrupt → เร็วขึ้น

---

### 3.7 บัส 8-bit vs 16-bit

ก. เร็วกว่า 2 เท่า  
ข. เฉลี่ย ~1.5 เท่า  

---

### 3.8 Daisy Chain Arbitration

ต้องใช้เวลาน้อยกว่า clock cycle  
เพื่อให้สัญญาณนิ่งก่อนตัดสิน

---

### 3.9 VAX SBI

อุปกรณ์สุดท้ายไม่ต้องมี TR  
เพราะได้สิทธิ์อัตโนมัติ

---

### 3.10 Priority ต่ำสุด

ได้เวลาเฉลี่ยต่ำสุด  
แต่ไม่จริงถ้ามี request จากตัวที่ priority สูงมาก

---

### 3.11 Memory Read Timing

Clock = 10 MHz → 100 ns  
Cycle ≈ 300 ns  

ต้องวางข้อมูลก่อน ~20 ns

---

### 3.12 Wait States

ก. ต้องเพิ่มประมาณ 2 cycles  
ข. Ready ต้อง LOW ระหว่างรอ

---

### 3.13 Memory Write

ก. Write width = T - 50 = 150 ns  
ข. Data valid รวม ≈ 170 ns  
ค. ต้องเพิ่ม wait state

---

### 3.14 Increment Instruction

ก. เพิ่มเวลา ~40–50%  
ข. ถ้า ALU ช้า → เพิ่มน้อยลง

---

### 3.15 Intel 8088

ก. ~1 MB/s  
ข. ลดลงครึ่งหนึ่งเมื่อมี wait state

---

### 3.16 Intel 8086

ใช้เวลา ~8–16 cycles สำหรับ operand 2 ตัว

---

### 3.17 32-bit Processor

ประสิทธิภาพเพิ่ม ~1.5–2 เท่า

---

### 3.18 Interrupt Timing

เกิดหลัง instruction ปัจจุบัน  
ประมาณหลายร้อย ns
